## 一、使用者：安装与启动

### Mac 用户

#### 方式 A：使用 `.app`（推荐）

1. 拿到 **`DeskKit.app`**（或从 `.dmg` 里拖出）
2. 双击打开
3. 若系统提示「无法打开，因为来自身份不明的开发者」：
   - **右键** `DeskKit.app` → **打开** → 再点 **打开**
   - 或：**系统设置 → 隐私与安全性** → 允许打开

#### 方式 B：使用 `.dmg`

1. 双击 `DeskKit_1.0.0_xxx.dmg`
2. 将 **DeskKit** 拖到 **应用程序** 文件夹
3. 从启动台或应用程序文件夹打开

> Tauri 版安装包约 **15～25MB**，无需再装 Node.js。

### Windows 用户

1. 拿到 **`DeskKit_1.0.0_x64-setup.exe`** 或 **`.msi`**
2. 双击安装，按向导下一步
3. 首次运行若提示 WebView2 相关错误，按提示安装 [Microsoft Edge WebView2 运行时](https://developer.microsoft.com/microsoft-edge/webview2/)（Win10/11 多数已自带）

### 启动后

- 窗口标题为 **DeskKit**，左上角有绿色 **Tauri** 标识
- 顶部导航切换功能模块
- **数据（路径、Token、进度等）保存在本机应用内**，不会上传到网络

## 二、使用者：界面与常用流程

### 界面说明

| 菜单位置 | 功能 |
|----------|------|
| 文件夹对比 | 多文件夹 MD5/路径对比、同步、删除、移动 |
| 批量重命名 | 前缀/后缀/替换/序号/清理非法字符 |
| 重复文件 | 同目录按 MD5 找重复 |
| 语雀导出 | 单篇或整库导出 Markdown / HTML |
| Confluence 转换 | 本地 `.md` 转 HTML / Word |
| 收藏管理 | 保存常用文件夹组合 |
| 设置 | 忽略规则、默认对比模式 |

每个页面右上角有 **「清除历史数据」**，可清掉该页记住的路径、Token 等。

选文件夹时，点输入框旁的 **文件夹图标**，在系统对话框里选择目录。

### 流程 1：语雀 → Confluence（推荐）

目标：把语雀知识库导出后，粘贴到 Confluence 编辑器。

1. 打开 **语雀导出**
2. **认证方式** 选 **API Token（推荐）**
   - 登录 [语雀 Token 设置](https://www.yuque.com/settings/tokens) 创建 Token，粘贴到输入框
3. **语雀知识库链接** 填：`https://www.yuque.com/你的用户名/知识库名`
4. **保存目录** 选一个空文件夹
5. **导出模式** 选「批量导出整个知识库」
6. **导出格式** 勾选 **Confluence 网页 (.html)**
7. **图片处理** 选 **「下载到本地 assets/」**
8. 点 **预览知识库** → **批量导出知识库**
9. 导出完成后，用浏览器打开生成的 `.html` → **全选复制** → 粘贴到 Confluence

> 不要把原始 Markdown 直接贴进 Confluence，容易报「无法保存页面」。

**防限流（可选）**：批量导出时可设固定或随机间隔（如 3～30 秒）；遇 `Too Many Requests` 等 5～10 分钟再继续，并勾选 **断点续导**。

---

### 流程 2：已有 Markdown → Confluence

1. 打开 **Confluence 转换**
2. 选择含 `.md` 文件的文件夹
3. 输出格式选 **html**
4. 勾选要转换的文件 → **批量转换**
5. 打开生成的 HTML，全选复制，粘贴到 Confluence

---

### 流程 3：对比并同步两个文件夹

1. 打开 **文件夹对比**
2. 添加至少 2 个文件夹，设一个为 **主文件夹（★）**
3. 对比模式选 **MD5**（看内容）或 **路径**（只看有无）
4. 点 **开始对比**
5. 勾选差异项 → **同步**（可先 **预览** 再确认）

---

### 流程 4：批量重命名

1. 打开 **批量重命名**
2. 选择目标文件夹
3. 在左侧配置规则（前缀、后缀、替换、序号等）
4. **预览** → 确认无误 → **执行**

---

### 流程 5：查找重复文件

1. 打开 **重复文件**
2. 选择目录 → **开始扫描**
3. 勾选要删的副本 → **删除选中**

## 三、开发者：环境准备

只需做一次。

### 1. 安装 Node.js

- 打开 [https://nodejs.org](https://nodejs.org)
- 下载 **LTS**，安装后终端验证：

```bash
node -v    # 建议 v24+
npm -v
```

### 2. 安装 Rust

Tauri 后端为 Rust，需安装工具链：

```bash
# 若未安装，执行（按官网提示）：
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 验证
rustc --version
cargo --version
```

### 3. Mac 额外依赖（打包用）

```bash
xcode-select --install
```

### 4. Windows 额外依赖（打包用）

- 安装 [Visual Studio Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/)，勾选 **「使用 C++ 的桌面开发」**
- 安装 [WebView2](https://developer.microsoft.com/microsoft-edge/webview2/)（运行时，开发机通常已有）

### 5. 获取源码并安装依赖

```bash
cd /Users/xiaojiang/web/deskKit    # 换成你的实际路径
npm install
```

首次 `npm run tauri:build` 会编译 Rust，耗时约 **3～10 分钟**，属正常现象。

## 四、开发者：本地运行

### 开发模式（改代码、热更新）

```bash
cd /Users/xiaojiang/web/deskKit
npm run tauri:dev
```

- 会启动 Vite 前端 + Tauri 窗口
- 改 Vue 代码保存后界面自动刷新
- 改 Rust 代码后需重启 `tauri:dev`

### 仅跑前端（调 UI，不连 Rust）

```bash
npm run dev
```

浏览器打开 `http://localhost:1420`，但 **文件/语雀等功能不可用**（需 Tauri 后端）。日常开发请用 `tauri:dev`。

## 五、开发者：打包安装包

### 命令速查

在项目根目录执行：

```bash
cd /Users/xiaojiang/web/deskKit

# ① 安装依赖（首次或 package.json 变更后）
npm install

# ② 打包当前系统架构（Mac 上默认 arm64）
npm run tauri:build
```

**`npm run tauri:build` 等价于：**

```bash
npm run build          # TypeScript 检查 + Vite 构建前端 → dist/
cd src-tauri && cargo tauri build
```

### 产物位置

| 平台 | 路径 | 文件 |
|------|------|------|
| macOS | `src-tauri/target/release/bundle/macos/` | `DeskKit.app` |
| macOS | `src-tauri/target/release/bundle/dmg/` | `DeskKit_1.0.0_aarch64.dmg`（若 dmg 脚本成功） |
| Windows | `src-tauri/target/release/bundle/msi/` | `.msi` 安装包 |
| Windows | `src-tauri/target/release/bundle/nsis/` | `-setup.exe` |

**直接发同事**：Mac 发 **`DeskKit.app`**（压缩成 zip 也行）或 **`.dmg`**；Windows 发 **`.exe` / `.msi`**。

**已打包产物（本地 release 目录）**：

打包完成后可将 dmg 复制到项目 `release/` 目录，便于统一分发：

```bash
./scripts/build-mac.sh   # 自动打 Mac 双架构并复制到 release/
```

| 文件 | 适用 |
|------|------|
| `release/DeskKit-1.0.0-mac-arm64.dmg` | Mac M 系列（M1/M2/M3） |
| `release/DeskKit-1.0.0-mac-intel.dmg` | Mac Intel 芯片 |
| `release/DeskKit-1.0.0-win-x64-setup.exe` | Windows（需在 Windows 或 CI 上打包） |

---

### Mac：Apple 芯片 + Intel 双架构

```bash
# 安装交叉编译 target（只需一次）
rustup target add aarch64-apple-darwin x86_64-apple-darwin

# 分别打包
npm run tauri:build:mac
```

等价于：

```bash
npm run build
npx tauri build --target aarch64-apple-darwin   # M 系列 Mac
npx tauri build --target x86_64-apple-darwin    # Intel Mac
```

产物示例：

- `DeskKit_1.0.0_aarch64.dmg` — M1/M2/M3
- `DeskKit_1.0.0_x64.dmg` — Intel Mac

---

### Windows 打包

**必须在 Windows 电脑上执行**（或在 CI 的 `windows-latest` 上）：

```bash
cd deskKit
npm install
npm run tauri:build
```

---

### 只打出 `.app`、跳过 dmg（dmg 失败时）

若 `tauri build` 最后报 `bundle_dmg.sh` 失败，**`.app` 通常已经生成**，可直接使用：

```bash
open src-tauri/target/release/bundle/macos/DeskKit.app
```

手动分发：右键 **DeskKit.app** → **压缩**，把 zip 发给同事。

## 六、发布新版本

发新版前：

1. 修改 **`src-tauri/tauri.conf.json`** 里的 `version`（如 `1.0.1`）
2. 同步修改 **`package.json`** 的 `version`
3. 重新打包：

```bash
npm run tauri:build
```

4. 将 `bundle/` 下新产物上传到 GitHub Releases 或发给用户

## 七、常见问题

### Mac 打不开 DeskKit.app

- 右键 → **打开**（不要用双击第一次）
- 或：系统设置 → 隐私与安全性 → 仍要打开

### 打包很慢 / 第一次很久

- 首次需下载并编译 Rust 依赖，正常
- 之后增量编译会快很多

### `tauri build` 报 Rust 相关错误

```bash
cd src-tauri
cargo check
```

根据报错安装缺失依赖（Mac 上常见是 Xcode Command Line Tools）。

### 语雀导出报 Too Many Requests

- 加大 **随机间隔**（如 3～30 秒）
- 等待 5～10 分钟再 **继续导出**（勾选断点续导）

### Confluence 粘贴后无法保存

- 用 **HTML 导出 + 浏览器全选复制**，不要直接贴 Markdown
- 检查内容是否含大量 emoji / 特殊字符
- 图片需选 **下载到本地 assets/**

### PDF 导出

Tauri 版 **不提供 Puppeteer PDF**。请：

1. 导出 **HTML**
2. 浏览器打开 → **打印** → **存储为 PDF**

### 清除所有本地记录

- 各页面右上角 **清除历史数据**
- 或在应用内 **设置** 页操作（设置项本身不会被清除）

### 与旧版 Electron 的区别

| 项目 | Electron 版 (`sync_file`) | Tauri 版 (`deskKit`) |
|------|---------------------------|----------------------|
| 使用者需 Node | 命令行版需要 | **不需要** |
| 安装包大小 | ~350MB | ~17MB |
| 启动方式 | `start.command` / 双击 app | 双击 **DeskKit.app** |
| PDF | 内置 Puppeteer | 用 HTML 打印为 PDF |
